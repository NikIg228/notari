# Сводка по созданным Wizard схемам

## Статус

✅ **Все 43 документа имеют созданные wizard схемы**

## Структура схем

Каждая схема включает следующие базовые шаги (в зависимости от типа документа):

### Общие шаги для большинства документов:

1. **Выбор количества доверителей** (если применимо)
   - Radio: 1 или 2

2. **Доверитель 1**
   - ФИО, ИИН, Дата рождения
   - Адрес
   - Документ (тип, серия/номер, кем выдан, дата выдачи)
   - Телефон (опционально)

3. **Доверитель 2** (условно, если выбрано 2)
   - Те же поля, что и для доверителя 1

4. **Выбор количества поверенных** (если применимо)
   - Radio: 1 или 2

5. **Поверенный 1**
   - ФИО, ИИН (опционально), Дата рождения
   - Адрес, Гражданство
   - Документ (тип, серия/номер, кем выдан, дата выдачи)
   - Телефон (опционально)

6. **Поверенный 2** (условно, если выбрано 2)
   - Те же поля, что и для поверенного 1

### Специфичные шаги по типам документов:

#### Документы с детьми (1, 2, 14, 39, 40, 42, 43):
- **Дети** (array, 1-10 элементов)
  - ФИО, ИИН, Дата рождения
  - Гражданство
  - Документ (тип, номер, дата выдачи, кем выдан)

#### Документы с выездом за границу (1, 18):
- **Выбор стран поездки** (checkbox)
  - Список всех стран + "Весь мир"

#### Документы с поездками по РК (2):
- **Маршрут внутри РК** (checkbox)
  - Список регионов РК + "Вся Республика Казахстан"

#### Документы с транспортными средствами (4, 5, 28):
- **Данные транспортного средства**
  - Тип ТС, Марка, Модель
  - Гос. номер, VIN
  - Год выпуска
  - Документ регистрации

#### Документы с ТМЦ (3):
- **Где и у кого забирать**
  - Наименование компании
  - БИН/ИИН, Адрес
  - Доп. данные

- **Что именно забрать (ТМЦ)** (array, минимум 1)
  - Наименование, Марка/модель
  - Количество, Единица измерения
  - Примечание

#### Документы со сроком действия:
- **Срок доверенности**
  - Тип срока (Дни/Месяцы)
  - Количество дней (1-31) или месяцев (1-36)

## Файлы схем

Все схемы находятся в директории `data/wizard-schemas/`:
- `1.json` - `43.json` (по одному файлу на каждый документ)

## Использование

Схемы автоматически загружаются при открытии страницы `/wizard/[id]` и используются компонентом `WizardEngine` для построения пошагового интерфейса.

## Следующие шаги

Для улучшения схем можно:
1. Добавить специфичные поля для каждого типа документа
2. Реализовать условную логику для сложных документов (договоры недвижимости)
3. Добавить валидацию специфичных полей
4. Создать специализированные компоненты для сложных блоков (недвижимость, договоры)

## Команды

```bash
# Регенерация всех схем
npm run generate-wizards

# Генерация documents.json из docs.txt
npm run generate-docs
```
